Spring boot JPA (Part-6) [OneToOne Mapping (Uni & Bidirectional)]

🔶 1. @OneToOne Mapping Overview

  • A @OneToOne relationship means one entity is associated with exactly one instance of another entity.
  • Examples:
    • A User has one Passport.
    • A UserDetails has one UserAddress.

🔸 2. ✅ Unidirectional @OneToOne Mapping

📌 Key Points:

  • Only one entity (owner) maintains reference to the other.
  • The foreign key is maintained in the owner table (Parent).
  • Use @JoinColumn to customize FK column.

✅ Example:

@Entity
@Table(name = "user_address")
public class UserAddress {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String street, city, state, country, pinCode;
    // Getters and Setters
}

@Entity
@Table(name = "user_details")
public class UserDetails {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name, phone;

    @OneToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "address_id", referencedColumnName = "id")
    private UserAddress userAddress;
    // Getters and Setters
}

Generated FK column in DB: address_id in user_details table.

🔸 3. ✅ Bidirectional @OneToOne Mapping

📌 Key Points:

  • Both entities reference each other.
  • One side owns the relationship, other uses mappedBy.
  • Avoid infinite recursion in JSON.

✅ Example:

@Entity
@Table(name = "user_details")
public class UserDetails {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name, phone;

    @OneToOne(cascade = CascadeType.ALL)
    @JoinColumn(name = "address_id", referencedColumnName = "id")
    @JsonManagedReference
    private UserAddress userAddress;
}

@Entity
@Table(name = "user_address")
public class UserAddress {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String street, city, state, country, pinCode;

    @OneToOne(mappedBy = "userAddress", fetch = FetchType.EAGER)
    @JsonBackReference
    private UserDetails userDetails;
}

🔸 4. ✅ Composite Key Mapping

📌 Use When:

  • FK is based on multiple fields (not a single ID).
  • Use @EmbeddedId and @Embeddable.

✅ Example:

@Embeddable
public class UserAddressCK {
    private String street;
    private String pinCode;
    // equals() and hashCode() required
}

@Entity
public class UserAddress {
    @EmbeddedId
    private UserAddressCK id;
    private String city, state, country;
}

@Entity
public class UserDetails {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;

    @OneToOne
    @JoinColumns({
        @JoinColumn(name = "address_street", referencedColumnName = "street"),
        @JoinColumn(name = "address_pinCode", referencedColumnName = "pinCode")
    })
    private UserAddress userAddress;
}

🔸 5. ✅ CascadeType in @OneToOne

CascadeTypeDescription
PERSISTSaving parent saves child automatically
MERGEUpdating parent updates child
REMOVEDeleting parent deletes child
REFRESHRefresh parent from DB, also child gets refreshed
DETACHRemove parent from persistence context, also detaches child
ALLShortcut for all of the above

🔸 6. ✅ FetchType in @OneToOne

Fetch TypeDescription
EAGERLoads child immediately with parent (default)
LAZYLoads child only when accessed (manually fetched)

✅ Fix for Lazy Loading + JSON:

  • Lazy loading causes 500 Internal Server Error on GET due to Hibernate proxy.
  • Solutions:
    • Use DTO to manually construct the response.
    • Use @JsonIgnore on child field (but you lose that data).
    • Use @JsonIdentityInfo to prevent infinite recursion.
    • Use @JsonManagedReference and @JsonBackReference in bidirectional mappings.

🔸 7. ✅ Serialization Solutions (Jackson)

🔁 Problem:

  • Infinite loop during serialization due to bidirectional mapping.

✅ Fixes:

  1. @JsonManagedReference / @JsonBackReference (for Owner/Inverse)
  1. @JsonIdentityInfo(generator = ObjectIdGenerators.PropertyGenerator.class, property = "id")
    • Prevents serialization of already seen objects.
  1. Use DTOs for full control.

🔸 9. ✅ Common Interview Questions

💬 What is the difference between unidirectional and bidirectional @OneToOne mapping?

Unidirectional: Only one entity (typically parent) has a reference to the other.Bidirectional: Both entities hold references. mappedBy is used on inverse side.

💬 What happens if you don’t use CascadeType?

The associated child entity won't be persisted/updated/deleted automatically.

💬 What’s the default fetch strategy for @OneToOne?

EAGER, unless explicitly set to LAZY.

💬 How do you avoid infinite recursion with bidirectional mapping?

Use @JsonManagedReference and @JsonBackReferenceOr use @JsonIdentityInfo to handle references by IDOr use DTO

💬 Can @OneToOne be mapped using a composite key?

Yes, using @EmbeddedId + @JoinColumns.
