Spring boot JPA (Part 7) [One-to-Many: Unidirectional]

🔶 1. @OneToMany Mapping (Unidirectional)

✅ Definition:

  • One entity (Parent) has many Child entities.
  • Example: A User can have many Orders.

📌 Default Behavior:

Feature Value
FK location New Join Table by default
Loading Strategy LAZY
Cascade Not enabled unless specified

✅ Example: Default (New Join Table)

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany
    private List<Order> orders = new ArrayList<>();
}

🔹 This will create a new join table like user_orders(user_id, orders_id).

✅ Avoiding Join Table with @JoinColumn

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany(cascade = CascadeType.ALL)
    @JoinColumn(name = "user_id") // FK goes into child (Order) table
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;
}

📝 Why @JoinColumn?

  • It avoids creating a separate join table.
  • FK (user_id) is directly present in Order table.

📦 Cascade Types:

Type Description
PERSIST Saving parent also saves all children
MERGE Updating parent updates children
REMOVE Deleting parent deletes children
ALL Includes all above

🧹 Orphan Removal:

@OneToMany(cascade = CascadeType.ALL, orphanRemoval = true)
@JoinColumn(name = "user_id")
private List<Order> orders;

🔹 If you remove an Order from user.getOrders(), it will be deleted from the DB too.

🔶 2. @OneToMany (Bidirectional)

✅ Features:

  • User holds List<Order>
  • Order holds a reference to User

✅ Example:

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

📝 mappedBy = "user" tells JPA: Order table holds the FK.

🔶 3. @ManyToOne (Unidirectional)

✅ Features:

  • Only child holds reference to parent.
  • Common for Order → User.

✅ Example:

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

🔶 4. @ManyToOne (Bidirectional)

  • Same as @OneToMany bidirectional from the other side.
  • Just add @OneToMany(mappedBy="user") in User.

🔶 5. @ManyToMany (Unidirectional)

✅ Features:

  • Each entity holds reference to the other.
  • Creates a new join table.

✅ Example:

@Entity
public class Product {
    @Id @GeneratedValue
    private Long id;
    private String name;
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String customer;

    @ManyToMany
    private List<Product> products = new ArrayList<>();
}

🔹 Creates: order_products(order_id, product_id)

🔶 6. @ManyToMany (Bidirectional)

✅ Example:

@Entity
public class Product {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @ManyToMany(mappedBy = "products")
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String customer;

    @ManyToMany
    @JoinTable(
        name = "order_products",
        joinColumns = @JoinColumn(name = "order_id"),
        inverseJoinColumns = @JoinColumn(name = "product_id")
    )
    private List<Product> products = new ArrayList<>();
}

🔶 7. EAGER vs LAZY Loading

Annotation Default Fetch Type
@OneToMany LAZY
@ManyToOne EAGER
@ManyToMany LAZY
⚠️ LAZY can cause LazyInitializationException outside of transaction. Fix using DTO or join fetch queries.

🔶 8. Summary Table

Mapping Direction FK Stored In New Table Use Case Example
OneToMany Unidirectional Join Table Yes User → Orders
OneToMany With FK Child Table No User → Orders with FK
OneToMany Bidirectional Child Table No User ↔ Orders
ManyToOne Unidirectional Child Table No Order → User
ManyToMany Unidirectional Join Table Yes Order → Products
ManyToMany Bidirectional Join Table Yes Order ↔ Products

🔶 9. Common Interview Questions

💬 How does @OneToMany work by default?

It creates a new join table, unless overridden with @JoinColumn.

💬 When should I use orphanRemoval = true?

When you want children deleted from the DB as soon as they are removed from the collection.

💬 What’s the difference between CascadeType.REMOVE and orphanRemoval?

CascadeType.REMOVE deletes children when parent is deleted.

orphanRemoval = true deletes children when removed from the collection (even without deleting parent).

💬 Can you have a unidirectional @OneToMany without a join table?

Yes, by placing the FK in the child using @JoinColumn.

💬 What's the default fetch type for @ManyToOne?

EAGER.
