üî∂ 1. @OneToMany Mapping (Unidirectional)

‚úÖ Definition:

  • One entity (Parent) has many Child entities.
  • Example: A User can have many Orders.

üìå Default Behavior:

FeatureValue
FK locationNew Join Table by default
Loading StrategyLAZY
CascadeNot enabled unless specified

‚úÖ Example: Default (New Join Table)

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany
    private List<Order> orders = new ArrayList<>();
}

üîπ This will create a new join table like user_orders(user_id, orders_id).

‚úÖ Avoiding Join Table with @JoinColumn

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany(cascade = CascadeType.ALL)
    @JoinColumn(name = "user_id") // FK goes into child (Order) table
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;
}

üìù Why @JoinColumn?

  • It avoids creating a separate join table.
  • FK (user_id) is directly present in Order table.

üì¶ Cascade Types:

TypeDescription
PERSISTSaving parent also saves all children
MERGEUpdating parent updates children
REMOVEDeleting parent deletes children
ALLIncludes all above

üßπ Orphan Removal:

@OneToMany(cascade = CascadeType.ALL, orphanRemoval = true)
@JoinColumn(name = "user_id")
private List<Order> orders;

üîπ If you remove an Order from user.getOrders(), it will be deleted from the DB too.

üî∂ 2. @OneToMany (Bidirectional)

‚úÖ Features:

  • User holds List<Order>
  • Order holds a reference to User

‚úÖ Example:

@Entity
public class User {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

üìù mappedBy = "user" tells JPA: Order table holds the FK.

üî∂ 3. @ManyToOne (Unidirectional)

‚úÖ Features:

  • Only child holds reference to parent.
  • Common for Order ‚Üí User.

‚úÖ Example:

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String item;

    @ManyToOne
    @JoinColumn(name = "user_id")
    private User user;
}

üî∂ 4. @ManyToOne (Bidirectional)

  • Same as @OneToMany bidirectional from the other side.
  • Just add @OneToMany(mappedBy="user") in User.

üî∂ 5. @ManyToMany (Unidirectional)

‚úÖ Features:

  • Each entity holds reference to the other.
  • Creates a new join table.

‚úÖ Example:

@Entity
public class Product {
    @Id @GeneratedValue
    private Long id;
    private String name;
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String customer;

    @ManyToMany
    private List<Product> products = new ArrayList<>();
}

üîπ Creates: order_products(order_id, product_id)

üî∂ 6. @ManyToMany (Bidirectional)

‚úÖ Example:

@Entity
public class Product {
    @Id @GeneratedValue
    private Long id;
    private String name;

    @ManyToMany(mappedBy = "products")
    private List<Order> orders = new ArrayList<>();
}

@Entity
public class Order {
    @Id @GeneratedValue
    private Long id;
    private String customer;

    @ManyToMany
    @JoinTable(
        name = "order_products",
        joinColumns = @JoinColumn(name = "order_id"),
        inverseJoinColumns = @JoinColumn(name = "product_id")
    )
    private List<Product> products = new ArrayList<>();
}

üî∂ 9. Common Interview Questions

üí¨ How does @OneToMany work by default?

It creates a new join table, unless overridden with @JoinColumn.

üí¨ When should I use orphanRemoval = true?

When you want children deleted from the DB as soon as they are removed from the collection.

üí¨ What‚Äôs the difference between CascadeType.REMOVE and orphanRemoval?

CascadeType.REMOVE deletes children when parent is deleted.

orphanRemoval = true deletes children when removed from the collection (even without deleting parent).

üí¨ Can you have a unidirectional @OneToMany without a join table?

Yes, by placing the FK in the child using @JoinColumn.

üí¨ What's the default fetch type for @ManyToOne?

EAGER.
