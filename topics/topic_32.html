üîÑ Spring Security Filter Chain (Core Concept)

Spring Security registers a chain of servlet filters that process every incoming HTTP request. These filters are automatically wired into your application via a FilterChainProxy.

‚úÖ Default Filters in the Chain (simplified):

Order Filter Class Purpose
1 SecurityContextPersistenceFilter Restores SecurityContext from session
2 UsernamePasswordAuthenticationFilter Handles login POST /login
3 BasicAuthenticationFilter Handles Basic Auth headers
4 BearerTokenAuthenticationFilter Handles JWT (if used)
5 ExceptionTranslationFilter Translates access exceptions
6 FilterSecurityInterceptor Performs authorization

‚úÖ You can add custom filters before or after any of these using:

‚úÖ What Is Stored in SecurityContext?

SecurityContext context = SecurityContextHolder.getContext();
Authentication auth = context.getAuthentication();  // Principal, Authorities, etc.

This object holds the authenticated user's details and authorities (roles).

üß¨ PasswordEncoder in Authentication Flow

  • Passwords are always stored hashed, never plain text.
  • During login:
    1. Raw password is hashed using the same encoder (e.g., BCrypt).
    1. Hashed password is compared to stored password.
@Bean
public PasswordEncoder encoder() {
    return new BCryptPasswordEncoder();
}

üîÑ Stateful vs Stateless Authentication

Mechanism Type Stores Session? Token/Session Needed?
Form Login Stateful ‚úÖ Yes Uses JSESSIONID cookie
Basic Auth Stateless ‚ùå No Sends creds in every req
JWT Stateless ‚ùå No Sends token in header
OAuth2 (Auth Code) Both Depends Access + Refresh Token
API Key Auth Stateless ‚ùå No Uses custom header

‚úÖ Authentication Mechanism Examples

1. Form Login

http.formLogin()
    .loginPage("/login")
    .defaultSuccessUrl("/home", true)
    .permitAll();

‚úÖ Used in traditional web apps with sessions.

2. Basic Authentication 

http.httpBasic()
    .and().csrf().disable();

‚úÖ Stateless; every request sends credentials via header Authorization: Basic ....

3. JWT Authentication

  • Add BearerTokenAuthenticationFilter
  • Validate JWT on each request
  • Stateless
http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);

4. OAuth2 Login (Social Login)

http.oauth2Login()
    .loginPage("/oauth2/authorization/google");

‚úÖ Supports GitHub, Google, Facebook, etc.

üí¨ Top Spring Security Interview Questions

Q1: What is the role of AuthenticationManager in Spring Security?

Authenticates the request by delegating to one or more AuthenticationProviders.

Q2: What is the difference between Authentication and Authorization?

Authentication = Identify the userAuthorization = Check user‚Äôs access/permissions

Q3: Can we secure endpoints based on roles?

Yes. Using:
@PreAuthorize("hasRole('ADMIN')")

Q4: What's the difference between stateful and stateless auth?

Stateful = session stored in server (cookie-based)

Stateless = no server session; token sent in every request (e.g., JWT)

Q5: Can we plug in custom authentication provider?

Yes. Implement AuthenticationProvider and register via config.
