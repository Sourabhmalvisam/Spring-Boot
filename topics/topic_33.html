Spring Security (Part-2 and Part-3)

✅ Form Login Authentication – Spring Security

🔐 1. What is Form Login Authentication?

  • It's a stateful authentication method where the server maintains user state via an HTTP session (usually JSESSIONID).
  • Used when you want to persist login across requests without re-entering credentials.

🧭 2. Form Login Flow – End-to-End

Client -> Sends POST request to /login (username & password)
↓
Security Filter Chain intercepts request
↓
UsernamePasswordAuthenticationFilter
↓
AuthenticationManager (Delegates to ProviderManager)
↓
DaoAuthenticationProvider
↓
UserDetailsService (loads user from DB or memory)
↓
PasswordEncoder (validates password - matches raw vs stored)
↓
If valid -> Authentication object is set in SecurityContext
↓
SecurityContext saved in HTTPSession (JSESSIONID)
↓
Response sent with Set-Cookie: JSESSIONID

📌 3. Default Configs

Feature Value
Default Login URL /login
Default Logout URL /logout
Default user user
Password (auto-generated) Printed in logs

You can override these via:

  • application.properties
  • Custom SecurityFilterChain

🧑‍💻 4. User Creation – 3 Ways

🟡 A. application.properties

spring.security.user.name=ganesh
spring.security.user.password={noop}pass123
  • {noop}: No hashing applied.

🟡 B. InMemoryUserDetailsManager Bean (Dev only)

@Bean
public UserDetailsService userDetailsService() {
    UserDetails user = User.withUsername("ganesh")
        .password("{noop}pass123")
        .roles("USER")
        .build();
    return new InMemoryUserDetailsManager(user);
}

🟢 C. Store in Database (Production)

  • Create UserAuthEntity implements UserDetails
  • Create UserAuthService implements UserDetailsService
  • Override loadUserByUsername()
  • Encode password using BCryptPasswordEncoder before storing

🧠 5. Password Encoding & Matching

🔁 Format:

{id}encodedPassword

Examples:

  • {noop}pass123
  • {bcrypt}$2a$10$...

🤖 Common Encoders:

Encoder Description
NoOpPasswordEncoder No hashing (dev use only)
BCryptPasswordEncoder Strong hashing with salt
DelegatingPasswordEncoder Delegates based on {id}

📘 6. Session Management

🔐 What is stored in session?

  • SecurityContext (contains Authentication object)

🔄 On next request:

  • Session ID (JSESSIONID) is sent in Cookie
  • Security Filter reads session → retrieves SecurityContext

🛑 Session Timeout:

  • Default: 30 mins
  • Can configure:
server.servlet.session.timeout=1m

💽 7. Store Session in DB (Spring Session JDBC)

  1. Add dependency:
<dependency>
  <groupId>org.springframework.session</groupId>
  <artifactId>spring-session-jdbc</artifactId>
</dependency>
  1. Add config:
spring.session.store-type=jdbc
spring.datasource.url=...
  1. Spring auto-creates SPRING_SESSION table

⚙️ 8. Session Creation Policy (important for interviews)

Policy Description
IF_REQUIRED Default. Creates session only when needed
ALWAYS Creates session always
NEVER Never creates, but can use existing
STATELESS No session at all (used in JWT APIs)

🔓 9. Permitting Public URLs

To allow /auth/register or /home:

http
  .authorizeHttpRequests()
  .requestMatchers("/auth/register", "/home").permitAll()
  .anyRequest().authenticated();

🔐 10. Authorization (Roles)

  • Role format: "ROLE_USER", "ROLE_ADMIN"
  • Spring appends ROLE_ prefix automatically for hasRole().
.antMatchers("/users").hasRole("USER")
  • Multiple roles per user supported.

🧪 11. CSRF in Form Login

  • Enabled by default in Spring Security (recommended)
  • Prevents cross-site request forgery attacks
  • Required hidden CSRF token in login forms:
<input type="hidden" name="_csrf" value="${_csrf.token}"/>

❗ 12. Common Drawbacks

Issue Description
❌ CSRF Vulnerability Must use CSRF protection
❌ Session Hijacking Cookies can be stolen
❌ Not scalable Session state needs shared storage in multi-node setups
❌ DB Load Storing sessions in DB can impact performance

📈 13. Real-World Best Practices

  • Always use BCrypt for password storage
  • Store sessions in distributed cache (Redis) if load balanced
  • Avoid {noop} and inMemory in production
  • Do not disable CSRF unless you know what you're doing
  • Customize login page using:
.formLogin().loginPage("/custom-login")
