üîê 1. What is Form Login Authentication?

  • It's a stateful authentication method where the server maintains user state via an HTTP session (usually JSESSIONID).
  • Used when you want to persist login across requests without re-entering credentials.

üßë‚Äçüíª 4. User Creation ‚Äì 3 Ways

üü° A. application.properties

spring.security.user.name=ganesh
spring.security.user.password={noop}pass123
  • {noop}: No hashing applied.

üü° B. InMemoryUserDetailsManager Bean (Dev only)

@Bean
public UserDetailsService userDetailsService() {
    UserDetails user = User.withUsername("ganesh")
        .password("{noop}pass123")
        .roles("USER")
        .build();
    return new InMemoryUserDetailsManager(user);
}

üü¢ C. Store in Database (Production)

  • Create UserAuthEntity implements UserDetails
  • Create UserAuthService implements UserDetailsService
  • Override loadUserByUsername()
  • Encode password using BCryptPasswordEncoder before storing

üìò 6. Session Management

üîê What is stored in session?

  • SecurityContext (contains Authentication object)

üîÑ On next request:

  • Session ID (JSESSIONID) is sent in Cookie
  • Security Filter reads session ‚Üí retrieves SecurityContext

üõë Session Timeout:

  • Default: 30 mins
  • Can configure:
server.servlet.session.timeout=1m

üíΩ 7. Store Session in DB (Spring Session JDBC)

  1. Add dependency:
<dependency>
  <groupId>org.springframework.session</groupId>
  <artifactId>spring-session-jdbc</artifactId>
</dependency>
  1. Add config:
spring.session.store-type=jdbc
spring.datasource.url=...
  1. Spring auto-creates SPRING_SESSION table

üîì 9. Permitting Public URLs

To allow /auth/register or /home:

http
  .authorizeHttpRequests()

  .anyRequest().authenticated();

üîê 10. Authorization (Roles)

  • Role format: "ROLE_USER", "ROLE_ADMIN"
  • Spring appends ROLE_ prefix automatically for hasRole().
.antMatchers("/users").hasRole("USER")
  • Multiple roles per user supported.

üß™ 11. CSRF in Form Login

  • Enabled by default in Spring Security (recommended)
  • Prevents cross-site request forgery attacks
  • Required hidden CSRF token in login forms:
<input type="hidden" name="_csrf" value="${_csrf.token}"/>

‚ùó 12. Common Drawbacks

Issue Description
‚ùå CSRF Vulnerability Must use CSRF protection
‚ùå Session Hijacking Cookies can be stolen
‚ùå Not scalable Session state needs shared storage in multi-node setups
‚ùå DB Load Storing sessions in DB can impact performance

üìà 13. Real-World Best Practices

  • Always use BCrypt for password storage
  • Store sessions in distributed cache (Redis) if load balanced
  • Avoid {noop} and inMemory in production
  • Do not disable CSRF unless you know what you're doing
  • Customize login page using:
.formLogin().loginPage("/custom-login")
