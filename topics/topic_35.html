Spring Security JWT(Json Web Token) Authentication

🔐 JWT Authentication – Spring Security (Stateless Auth)

✅ 1. What is JWT Authentication?

  • JWT is a Stateless Authentication mechanism.
  • Server does NOT maintain session state (no JSESSIONID).
  • Client sends JWT token with every request in the Authorization header.

🧱 2. JWT Structure

A JWT has 3 parts:

HEADER.PAYLOAD.SIGNATURE
Part Contains
Header Metadata: algorithm (e.g., HS256, RS256)
Payload Claims: userId, roles, expiry time, etc.
Signature Signed hash to prevent tampering

🔁 3. JWT Authentication Flow

🧾 Step-by-step Breakdown:

1️⃣ User Registration (optional)

POST /api/user-register

→ User created and password hashed (e.g., using BCryptPasswordEncoder).

2️⃣ Token Generation

POST /generate-token

→ User sends username/password

→ If valid: Generate JWT token and return in response

3️⃣ Access Resource

Client sends JWT in every request:

Authorization: Bearer <jwt_token>

→ Token is verified and user is authenticated

4️⃣ Refresh Token

POST /refresh-token

→ Send refresh token

→ Get new access token

⚙️ 4. Key Components to Implement

✅ Token Generation:

  • Authenticate using UsernamePasswordAuthenticationToken
  • Use AuthenticationManager and DaoAuthenticationProvider
  • On success → generate token using JWT utility class

✅ Token Validation:

  • Create Custom Filter to intercept each request
  • Parse JWT from Authorization header
  • Validate:
    • Expiry
    • Signature
    • Claims
  • If valid:
    • Load UserDetails
    • Create Authentication object
    • Set in SecurityContext

✅ Filter Order:

http.addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

🔧 6. AuthenticationManager Logic

  • Delegates to multiple AuthenticationProviders
  • Each Provider must implement supports() method to indicate whether it can handle the request
  • In JWT, we use:
    • Default DaoAuthenticationProvider for username/password login
    • Custom JwtAuthenticationProvider for token validation

🛡️ 7. Authorization

  • After authentication, AuthorizationFilter ensures role-based access.
  • Same as form or basic auth.
  • E.g.,
http.authorizeHttpRequests()
  .requestMatchers("/admin/**").hasRole("ADMIN")
  .requestMatchers("/user/**").hasAnyRole("ADMIN", "USER");

🔑 8. JWT Token Placement in Request

Authorization: Bearer <access_token>

📂 9. JWT Implementation – What Spring Boot does NOT provide

Spring Security doesn't include JWT by default because:

  • Token structure is app-specific
  • Signing strategy may vary (HMAC vs RSA)
  • Claims (e.g., email, userId) differ
  • Refresh strategies vary per use case
⚠️ You must implement it using custom filters, providers, utilities.

🧠 10. Advantages of JWT

Feature Benefit
✅ Stateless No need to store session
✅ Scalable Suitable for distributed systems
✅ Secure Signed & verifiable (if using HTTPS)
✅ Portable Can be used across services/systems
✅ Flexible Custom claims supported

🚫 11. Disadvantages of JWT

Problem Why
❌ Can’t invalidate token easily Unless using a token blacklist or DB
❌ Can be hijacked If HTTPS not enforced
❌ Size overhead Bigger than session ID
❌ Short-lived tokens needed For security, which needs refresh logic

🔁 12. Refresh Token Strategy

Token Type Duration Stored in
Access Token Short-lived (e.g., 5-15 mins) Header
Refresh Token Long-lived (e.g., 7-30 days) DB or Secure Cookie
Refresh token is used to generate new access token without login.

🧪 14. Sample Token Generation Code

String token = Jwts.builder()
  .setSubject(user.getUsername())
  .claim("role", user.getRole())
  .setIssuedAt(new Date(System.currentTimeMillis()))
  .setExpiration(new Date(System.currentTimeMillis() + 1000 * 60 * 60)) // 1 hour
  .signWith(SignatureAlgorithm.HS256, SECRET_KEY)
  .compact();
