🔐 OAuth 2.0 & OIDC – Interview-Ready Notes

✅ 1. What is OAuth 2.0?

  • OAuth 2.0 is an Open Authorization framework.
  • It enables secure, delegated access to protected user resources.
  • Used when a user wants to grant a third-party app access to their resources (e.g., Google Calendar, GitHub profile).

🔁 2. OAuth2 vs OIDC (OpenID Connect)

🔢 S.No Feature OAuth2 OIDC (OpenID Connect)
1️⃣ Full Form Open Authorization OpenID Connect
2️⃣ Used For Authorization Authentication + Authorization
3️⃣ Tokens Returned Access Token (opaque or JWT) ID Token (JWT) + Access Token
4️⃣ Identity Support ❌ No identity verification ✅ Verifies user identity via ID token
5️⃣ Mandatory Scope read, write, etc. openid (indicates OIDC use)
6️⃣ Protocol Type Authorization Framework Identity Layer over OAuth2

📥 3. Common Grant Types

Grant Type Description
Authorization Code Standard flow for web apps (secure, uses code + secret)
Implicit For SPAs, deprecated due to security flaws
Client Credentials Server-to-server communication
Password Grant Not recommended; app gets direct access to user credentials
Refresh Token Used to get a new Access Token when the current one expires

🔒 4. OAuth 2.0 – Authorization Code Flow (Step-by-Step)

🧠 Actors:

  • Resource Owner – the User
  • Client – your Spring Boot App
  • Authorization Server – GitLab, Auth0, Google, etc.
  • Resource Server – API server that holds protected data

✅ Step 1: Client Registers With Authorization Server

  • Receives:
    • client_id
    • client_secret
    • Redirect URI

✅ Step 2: User Clicks Login

GET /oauth2/authorization/{registrationId}

Spring Security generates login page using:

DefaultLoginPageGeneratingFilter

✅ Step 3: User is Redirected to Authorization Server

https://auth0.com/oauth/authorize?client_id=...&redirect_uri=...

✅ Step 4: Authorization Server Authenticates User & Returns Authorization Code

GET /login/oauth2/code/{registrationId}?code=abc123

Handled by:

OAuth2LoginAuthenticationFilter

✅ Step 5: Spring Boot Calls Token URI

  • Sends:
    • Authorization Code
    • Client ID & Secret
  • Receives:
    • Access Token (Opaque or JWT)
    • ID Token (JWT)
    • Refresh Token (optional)

Token processing by:

OidcAuthorizationCodeAuthenticationProvider

✅ Step 6: Spring Boot Creates Authentication Object

  • Sets user details (from ID Token or UserInfo endpoint)
  • Stores in:
SecurityContextHolder
HttpSession
InMemoryOAuth2AuthorizedClientService

🎯 5. ID Token vs Access Token

Feature Access Token ID Token (OIDC)
Format Opaque or JWT JWT
Purpose Authorize access to resource server Identify the user
Who Consumes Resource Server Client App
Contains Scopes, expiry, audience, etc. User info like email, sub, name, etc.

⚙️ 6. Spring Boot Setup (OIDC Integration)

pom.xml

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-oauth2-client</artifactId>
</dependency>

application.properties

spring.security.oauth2.client.registration.auth0.client-id=xxxx
spring.security.oauth2.client.registration.auth0.client-secret=yyyy
spring.security.oauth2.client.registration.auth0.scope=openid,profile,email
spring.security.oauth2.client.provider.auth0.issuer-uri=https://dev-xxxx.us.auth0.com/

💡 7. Why Postman Calls Fail?

Spring Boot assumes browser-based flow → creates a session.

So Postman calls fail since no cookie (JSESSIONID) is present.

🔄 8. Make OAuth2 Flow Stateless

  • Override onAuthenticationSuccess() of OAuth2LoginAuthenticationFilter
  • Return ID Token in response body (instead of creating session)
  • In subsequent requests, client must send:
Authorization: Bearer <id_token>
  • Use a custom filter to validate token (decode JWT, check expiry, signature, etc.)

🔐 9. Token Validation Flow (Stateless Setup)

Step Action
1️⃣ Intercept all requests with custom filter
2️⃣ Read Authorization: Bearer <token> header
3️⃣ Validate JWT (signature, expiry, issuer, audience)
4️⃣ If valid, load user & set Authentication object in SecurityContext
5️⃣ Proceed with request as authenticated user

⚖️ 10. SecurityContextHolder in OAuth2

Type Description
Holds the Authentication object
Used across the request lifecycle
If using sessions → backed by HttpSessionSecurityContextRepository
In stateless JWT setup → populated manually via custom filter

📥 11. Refresh Token Flow

  • Access token is short-lived (5–15 min)
  • Refresh token is long-lived (7–30 days)
  • Client sends refresh token to Authorization Server:
POST /oauth/token
grant_type=refresh_token
refresh_token=abc123
  • Gets new access token (and optionally new refresh token)

⚠️ 12. Common Pitfalls

Problem Cause
Postman access fails No session/cookie – needs stateless setup
ID Token expired Still have active session
Access Token is opaque Can’t decode client-side – need introspection endpoint
Using stateful auth in distributed apps Avoid – always go stateless for OAuth

🧪 13. Tools for Debugging

  • Use jwt.io to decode ID tokens
  • Use Auth0’s debugger tools to simulate flows
  • Look at Spring Security logs – filters and token processors

✨ 14. Final Summary – Interview Answers

Question Ideal Answer
What is OAuth2? Open authorization protocol used to delegate access without sharing credentials.
What is OIDC? Identity layer built on top of OAuth2, enables user authentication.
What is ID Token? JWT issued to client with user identity claims, used for authentication.
How does Spring Security support OAuth2? Via auto-configured filters and client registration in application.properties.
How to make it stateless? Return ID token, validate in each request via custom filter, avoid sessions.
What is registration_id? The alias for each provider defined in application.properties (e.g., auth0).
