Role based Authorization - (Annotation)

🔐 Role-Based Authorization Using Annotations in Spring Security

✅ 1. What is Role-Based Authorization?

Authorization is the process of allowing or denying access to specific resources based on the user’s role or permission.

Spring Security supports multiple ways to implement authorization, including:

  • Filter-level access control (via .antMatchers().hasRole())
  • Annotation-based (method-level) access control → this is our focus

🎯 2. Annotations Used

Annotation Description
@PreAuthorize Runs before method execution
@PostAuthorize Runs after method execution, but before response is sent

🔧 3. Enabling Annotation-Based Authorization

Add this in your security config:

@EnableMethodSecurity  // Replaces @EnableGlobalMethodSecurity in Spring Security 6+
public class SecurityConfig {
    // other beans & filters
}
⚠️ Without this, @PreAuthorize and @PostAuthorize annotations will be ignored.

🧪 4. Example Usage

👤 User with Roles and Permissions

User.withUsername("ganesh")
    .password("{noop}pass123")
    .roles("USER")  // Spring internally stores it as ROLE_USER
    .authorities("ORDER_READ", "SALES_DELETE")  // granular permissions
    .build();

🧾 OrderController

@RestController
public class OrderController {

    @PreAuthorize("hasRole('USER') and hasAuthority('ORDER_READ')")
    @GetMapping("/orders")
    public String getOrders() {
        return "Order Data";
    }
}

🧾 SalesController

@RestController
public class SalesController {

    @PreAuthorize("hasAuthority('SALES_READ')")
    @GetMapping("/sales")
    public String getSales() {
        return "Sales Data";
    }
}

🔍 5. hasRole vs hasAuthority

Method Meaning
hasRole('ADMIN') Looks for ROLE_ADMIN
hasAuthority('ADMIN') Looks for exact string ADMIN
⚠️ Internally, hasRole('X') appends "ROLE_" prefix.

💡 6. Expression Language Support (SpEL)

Spring uses SpEL (Spring Expression Language) to evaluate conditions in @PreAuthorize.

✅ Logical Operators

Operator Description Example
and Logical AND hasRole('ADMIN') and hasAuthority('READ')
or Logical OR hasRole('ADMIN') or hasRole('USER')
not / ! Logical NOT !hasRole('ADMIN')

✅ Relational Operators

Operator Meaning Example
== Equal to #id == 100
!= Not equal to #value != 'xyz'
< > <= >= Comparison operators

🧠 7. Accessing Principal Info (Dynamic Authorization)

You can access current user’s info using the authentication object:

@PreAuthorize("#id == authentication.principal.id")
@GetMapping("/users/{id}")
public User getUserById(@PathVariable Long id) {
    return userService.getUser(id);
}

⏳ 8. Post Authorization – @PostAuthorize

Use when you need to authorize based on return object or post-processing logic.

@PostAuthorize("returnObject.username == authentication.name")
@GetMapping("/user/{id}")
public User getUser(@PathVariable Long id) {
    return userService.findById(id);
}
✅ Example: Prevent user from accessing someone else’s data.

⚙️ 9. How Does It Work Internally?

Interceptors:

  • @PreAuthorize is intercepted by: 
    AuthorizationManagerBeforeMethodInterceptor
  • @PostAuthorize is intercepted by: 
    AuthorizationManagerAfterMethodInterceptor

They:

  1. Parse the annotation string using SpelExpressionParser
  1. Resolve symbols (like authentication.principal)
  1. Create an Abstract Syntax Tree (AST)
  1. Evaluate the expression
  1. Allow or block method execution

🧪 10. Real-World Scenario

Role Permissions
ROLE_USER ORDER_READ, ORDER_WRITE
ROLE_ADMIN All permissions

Sample:

@PreAuthorize("hasRole('USER') and hasAuthority('ORDER_WRITE')")
@PostMapping("/orders")
public String createOrder() {
    return "Created";
}

❌ 11. Common Mistakes

Mistake Fix
Not enabling @EnableMethodSecurity Add it in your config
Using hasRole('ADMIN') but setting authority ADMIN Use ROLE_ADMIN or use hasAuthority
Using @PreAuthorize on constructor or static method Not allowed
Trying to use request-specific data not available in context Use #paramName properly
