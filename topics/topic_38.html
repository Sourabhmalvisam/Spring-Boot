🔁 2. Inter-Service Communication

🔄 Communication Types:

  • Synchronous: Waits for response (blocking)
  • Asynchronous: Doesn’t wait (non-blocking; e.g., Kafka/RabbitMQ)
🧠 This guide focuses on Synchronous Communication

🔌 3. Synchronous Communication Options in Spring Boot

Method Notes
RestTemplate Legacy, still widely used
RestClient New (Spring 6), more concise
FeignClient Declarative HTTP client with built-in load balancing (Spring Cloud)

🧰 4. Using RestTemplate (Legacy, but Interview Favorite)

Example:

RestTemplate restTemplate = new RestTemplate();

String url = "http://localhost:8082/products/1";

Product product = restTemplate.getForObject(url, Product.class);

Internals of getForObject():

  • Uses HttpURLConnection under the hood via SimpleClientHttpRequestFactory
  • Performs a 3-way TCP handshake
  • Sends the request
  • Waits for response
  • Uses Keep-Alive cache to reuse TCP connections

🔍 6. What is Keep-Alive?

  • Default in HTTP/1.1
  • Reuses TCP connection for multiple HTTP requests
  • Controlled by headers: 
    Connection: keep-alive
Keep-Alive: timeout=5, max=50
  • Saves time from repeated handshakes

⚙️ 7. Methods in RestTemplate

Method Description
getForObject() Get only body
getForEntity() Get full response (headers + status + body)
postForObject() POST + body
postForEntity() POST + full response
put() Sends PUT request (no return)
delete() Sends DELETE request
exchange() Most flexible – allows headers, body, method
execute() Full manual control using callback interfaces

🧪 exchange() Example:

HttpHeaders headers = new HttpHeaders();
headers.set("Authorization", "Bearer abc123");

Product product = new Product("Ice-cream", 100);
HttpEntity<Product> requestEntity = new HttpEntity<>(product, headers);

ResponseEntity<Product> response = restTemplate.exchange(
    "http://localhost:8082/products",
    HttpMethod.POST,
    requestEntity,
    Product.class
);

🛠️ 8. Advanced: Manual execute() with Custom Callbacks

RequestCallback requestCallback = request -> {
    request.getHeaders().setContentType(MediaType.APPLICATION_JSON);
    byte[] body = new ObjectMapper().writeValueAsBytes(new Product("Ice-cream", 100));
    StreamUtils.copy(body, request.getBody());
};

ResponseExtractor<String> responseExtractor = response ->
    StreamUtils.copyToString(response.getBody(), StandardCharsets.UTF_8);

String response = restTemplate.execute(
    "http://localhost:8082/products",
    HttpMethod.POST,
    requestCallback,
    responseExtractor
);
✅ Gives full control over headers, serialization, and parsing.

🛑 9. Limitations of RestTemplate

Issue Why
🧱 Boilerplate-heavy Too many overloaded methods
❌ No built-in retries Retry/CircuitBreaker not built-in
⚠️ Low readability Verbose and imperative
🛑 Maintenance mode Deprecated for RestClient in Spring 6+

🚀 10. Why Move to RestClient?

Feature Benefit
Fluent Builder Clean and chainable
Built-in Config Headers, timeouts, error handlers
Future-proof Supports reactive & interceptors
✅ Recommended for Spring Boot 3+ / Spring Framework 6+

🧠 Interview Key Points

Question Must-Know
How does RestTemplate work internally? Uses HttpURLConnection, KeepAliveCache, TCP reuse
When is TCP connection closed? If response stream isn’t fully read or Connection: close is sent
How to reuse TCP? Use Connection: keep-alive (default in HTTP/1.1)
When to use exchange() vs execute()? exchange() is semi-automatic, execute() is fully manual
What’s better than RestTemplate in modern Spring? RestClient or FeignClient with Spring Cloud
How to handle headers and body together? Use HttpEntity<> in exchange()
